4.8. 函数定义详解

在Python中，抽象基类（Abstract Base Class，简称ABC）是一种特殊的类，它不能被直接实例化，而是用于定义其他类应该具有的共同接口或行为。抽象基类的主要目的是强制子类实现某些特定的方法，从而确保子类具有一致性和兼容性。

在Python中，abc模块提供了创建抽象基类的支持。要使用抽象基类，你需要从abc模块中导入ABC（表示抽象基类）和abstractmethod（表示抽象方法）。

以下是一个简单的示例，展示了如何使用抽象基类：
from abc import ABC, abstractmethod

# 定义一个抽象基类
class Animal(ABC):

    # 定义一个抽象方法
    @abstractmethod
    def speak(self):
        pass

# 定义一个具体的子类，它必须实现Animal类中的抽象方法
class Dog(Animal):

    def speak(self):
        return "Woof!"

# 另一个具体的子类
class Cat(Animal):

    def speak(self):
        return "Meow!"

# 创建一个Animal类的实例会导致TypeError，因为Animal是一个抽象基类
# animal = Animal()

# 创建具体的子类实例
dog = Dog()
cat = Cat()

# 调用子类的方法
print(dog.speak())  # 输出 "Woof!"
print(cat.speak())  # 输出 "Meow!"



抽象基类的优点包括：

强制一致性：确保所有子类都实现了必要的接口，这有助于保持代码的整洁和一致。

提高可读性：通过清晰地定义接口，抽象基类使得其他开发者更容易理解类的目的和用法。

易于维护：如果需要在未来修改接口，只需修改抽象基类，而无需逐个修改所有子类。

灵活性：抽象基类允许你根据需求添加新的子类，只要它们满足基类的要求即可。
